CSS Animation

CSS Animations are a cool and neat way to make interesting things happen on a page. All you need is some content and Excellence (also @ rules)

In this example, the above div has the class of .animation-example and in the CSS file is followed by @keyframes example-1. The property animation-name or the first word in the animation shorthand in the class definition binds the similarly named @keyframes rule to our div.

.animation-example{
width:75%;
margin:2em auto;
padding:1em;
border:3px groove gainsboro;
animation: example-1 5s linear 1s infinite alternate; }

@keyframes example-1{
0% { transform: scale(1); background-color: #EF8354; } 20% { transform:scale(0.8); } 40%{ transform:scale(0.4); background-color: firebrick; } 80%{ transform:scale(0.2); } 100%{ transform:scale(0); background-color: #AA8888; } }

.animation-example-two{
width:75%; position: relative; display: inline-block; padding:1em; border:3px groove firebrick; animation: example-2 8s 4s ease infinite alternate; }

@keyframes example-2{ 0% { right:0px; background-color: orange; } 50%{ right:-200px; background-color:darkslategray; border-radius: 50%; } 100%{ right:400px; background-color:lightgray; color:black; } }

Keyframes can be defined as From and To instead of percentages. Animations can play back and forth with the direction property instead of defining keyframes that move back to the original state.

Animation-Play-State property can be set to play or pause, allowing an animation to be started and stopped by input like :hover or a js event.

Animation-Fill-State specifies if the animation styles are visible before or after the animation plays. This property is a little confusing.

Animation shorthand property values:
  1. The first shorthand value is animation-name, and binds a corresponding @keyframes rule.
  2. The next value is Animation-Duration. In seconds, how long the element should take to cycle through its keyframes.
  3. The third value is Animation-Timing-Function. This specifies the animation curve for intermediary values. Set to ease by default, which starts slow but speeds up.
  4. The fourth value is Animation-Delay. This specifies a waiting time between load and animation start.
  5. The fifth value is Animation-Iteration-Count. This specifies how many times the animation should run.
  6. The sixth value is Animation-Direction. Normal by default but can be set to alternate or reverse.
